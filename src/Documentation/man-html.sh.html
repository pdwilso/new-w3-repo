HTML Manual Pages

The man(1) command in GNU/Linux can be used to reformat system manual pages as HTML documents.

The HTML formatting functionality in man(1) is provided by groff(1), the user interface to the GNU Document Formatting System.

  1. Check to see if groff(1) is installed:
    sudo apt show groff
  2. Install groff(1):
    sudo apt install groff
  3. Install Lynx web browser:
    sudo apt install lynx

For the following command(s) to work, both the groff(1) package and the Lynx web brower (lynx(1)) package must be installed.

Once those pre-requisites are installed, the following command may be used to produce an HTML redering of the specified man page:

man --html="lynx --source" groff

Note that under a default configuration for Debian GNU/Linux, with all packages installed using the apt toolset, the above command will by default run Lynx with the manual page loaded as an HTML document, unless one supplies the --source parameter to Lynx in the command shown above.

The man --html command shown above using the --source parameter to Lynx will dump the manual page HTML to stdout – redirection can be used to capture or further process the HTML, and the command may be scripted to provide real-time manual page conversion to HTML, on-the-fly by exec'ing the man command. 

#!/bin/bash
MAN_CMD_NAME = $1;
MAN_CMD_SECT = $2;
WWW_BROWSER_CMD="lynx --source"
MAN='man';
MAN_OPTS='--html="$WWW_BROWSER_CMD"'
$MAN $MAN_OPTS $MAN_CMD_NAME > $MAN_CMD_NAME.$MAN_CMD_SECT.html
