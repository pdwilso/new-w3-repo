sum codez: workflow

  1. Create and edit files on the Raspberry Pi over the LAN

  2. Create index.html files for certain HTML_DOCUMENT_ROOT sub-diretories.

    The document mk-indexes.pl.html provides a more detailed discussion of this process, and suggestes some example Perl code for automating the process.

    Note that list of the sub-directories to raspberrypi.local:/var/www/html/ may be found at ./directories on this server.

    Further note that not all the subdirectories listed are, nor should be, subject to automated index file construction as part of the automated site update process.

    As of this writing there is no definitive list of which directories may be safely added to a list for auto-updates of index files. 2023-06-18 03:13 EST

    /file-listing.php?fn=./forkls.php

  3. Use rsync to mirror rPi site up to Google Drive shared directory. The shared directory is typically mounted at ~/google-drive/ on the Raspberry Pi file system (as currently configured), so we're doing all this from the Raspberry Pi file system, for the time being.

    Some additional information about the process of updating file systems using rsync may be found on this server in the document sum codez: use rsync to update website

  4. Update GitHub repository pdwilso / sumkittehz-codes from the Google Drive shared directory. THis can be done from the Raspberry Pi, OR from any other workstation that has the Google Drive shared directory mounted at an accessible location on their local file system.
  5. Initiate a "clone repository" operation (from the GitHub repository, above) to the W3 Spaces hosted domain location, https://sumkittehz.codes/.

There are a number of ways this workflow can be optimized or modified as needed to fit particular circumstances. Some of the more obvious alternatives include using rsync over the nework via SSH, using e.g. rsync raspberrypi.local:/var/www/html/ syntax.
