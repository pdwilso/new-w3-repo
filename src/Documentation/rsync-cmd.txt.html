Using rsync to Transfer HTML_DOCUMENT_ROOT

In order to do this remotely it is pre-requisite that you have both an account on the remote host and that you have SSH working to connect across the network to the host where you are updating the site files.

We are using a Raspberry Pi 3b+ connected to our LAN via Ethernet. We have account login credentials established no the Pi, and will use ssh for all command line operations with the Pi, which is running "headless".

This document will describe how to use the rsync command line (CLI) tool to transfer the site content files from HTML_DOCUMENT_ROOT of a default Apache httpd install under the Debian-derived Raspberry Pi OS.

This is the command to update a target website from the HTML_DOCUMENT_ROOT of the time of this writing, 2023-06-15 18:00 EST. 


	  rsync -n
	  --exclude "*~"
	  --exclude "*.zip"
	  --exlude "*.pl" -avuzb --del
	  /var/www/html/ \$1
